> It connects to **any backend** via QR code pairing — no native SDK, no database, no setup.

## Overview

CamLiveAI does one thing: **capture camera frames → send to your API → display detections**.

All project-specific configuration comes from a **QR code** that the user scans. The QR
code contains everything CamLiveAI needs: your API URL, an auth token, and tenant metadata.

This means **any backend** that implements one POST endpoint can use CamLiveAI as its
mobile camera client. No app store, no build pipeline, no native code.

### What CamLiveAI Provides

- Full-screen camera feed with `getUserMedia`
- QR scanner (auto-starts, powered by `jsQR`)
- Real-time frame capture (single / live / batch modes)
- Bounding box overlay on video
- Detection feed with confidence colors
- Settings (camera source, confidence threshold, grid overlay)
- User panel with connection info and disconnect

### What Your Backend Must Provide

- One `POST /detection/single` endpoint (or equivalent)
- JWT auth token embedded in the QR code
- CORS allowing requests from `https://camliveai.netlify.app`

## Connection Flow

```
┌──────────┐     QR Code JSON      ┌──────────┐
│  Your    │ ────────────────────→ │ CamLiveAI│
│ Project  │  {token, apiUrl, ...} │   App    │
└──────────┘                      └────┬─────┘
                                       │
                              (auto-starts camera)
                                       │
                              ┌────────▼────────┐
                              │  Capture frame   │
                              │  → base64 JPEG   │
                              └────────┬────────┘
                                       │
                              POST /detection/single
                              Authorization: Bearer <token>
                              Body: { image: "<base64>" }
                                       │
                              ┌────────▼────────┐
                              │  Your Backend    │
                              │  → YOLO / CV     │
                              └────────┬────────┘
                                       │
                              Response: { detections: [...] }
                                       │
                              ┌────────▼────────┐
                              │  Render bboxes   │
                              │  on camera feed  │
                              └─────────────────┘
```

### Step-by-Step

1. User opens `https://camliveai.netlify.app`
2. CamLiveAI auto-starts the rear camera for QR scanning
3. User points at a QR code generated by your project
4. `jsQR` decodes the QR → parses JSON → validates required fields
5. CamLiveAI switches to full-screen camera mode
6. Frames are captured at the configured interval:
   - **Single**: every 3 seconds
   - **Live**: every 500ms
   - **Batch**: 3 frames every 5 seconds
7. Each frame is sent as base64 JPEG to your API
8. Your API returns detections → CamLiveAI draws bounding boxes

## QR Code Format

The QR code must encode a **JSON object** with these exact fields:

```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "tenantType": "agriculture",
  "tenantSlug": "fruitsight",
  "cameraId": "cam-001",
  "apiUrl": "https://fruitsight-ai.onrender.com"
}
```

### Field Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `token` | `string` | ✅ | JWT or API token for `Authorization: Bearer` header |
| `tenantType` | `string` | ✅ | Domain identifier (see [Domain Types](#domain--tenant-types)) |
| `tenantSlug` | `string` | ✅ | Human-readable project name slug (displayed in UI) |
| `cameraId` | `string` | ✅ | Unique camera/device identifier for your records |
| `apiUrl` | `string` | ✅ | Base URL of your detection API (no trailing slash) |

### Validation Rules

CamLiveAI validates the QR data. The scan is accepted only when:
- `token` is a non-empty string
- `tenantType` is a non-empty string
- `cameraId` is a non-empty string
- JSON parses successfully

If validation fails, the scanner silently continues.

### QR Generation Tips

- QR code version: **3-5** (29×29 to 37×37) is sufficient for ~200 bytes
- Error correction: **M** (15%) or **Q** (25%) for reliable scanning from phone screens
- Encode the **exact** JSON — no extra whitespace if possible (smaller QR = faster scan)
- Test with your phone's camera before deploying

## Backend API Contract

### `POST /detection/single`

Your project must expose one endpoint that:

- Accepts `POST` requests
- Parses JSON body with a base64-encoded JPEG image
- Returns detections in the expected response format

#### Request

```
POST /detection/single
Content-Type: application/json
Authorization: Bearer <token-from-qr>
```

```json
{
  "image": "<base64-encoded-jpeg>",
  "confidence": 0.5,
  "iou": 0.45
}
```

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `image` | `string` | ✅ | — | Base64-encoded JPEG (no `data:` prefix) |
| `confidence` | `number` | ❌ | `0.5` | Confidence threshold (0-1) |
| `iou` | `number` | ❌ | `0.45` | IoU threshold for NMS |

**Note about `image` format:** CamLiveAI strips the `data:image/jpeg;base64,` prefix
before sending. If your API handles both formats, add a check for the `data:` prefix.

#### Response (Success 200)

```json
{
  "detections": [
    {
      "bbox": [100, 50, 200, 180],
      "class_name": "apple",
      "confidence": 0.92,
      "class_id": 47,
      "grade": "premium"
    }
  ],
  "total_count": 1,
  "processing_time_ms": 1620.5,
  "model_used": "agriculture",
  "tenant_type": "agriculture"
}
```

#### Response (Error 4xx/5xx)

CamLiveAI logs errors to console silently — it never shows error modals to the user.
It continues capturing frames regardless.

## Response Format

### Detection Object

Each detection in the `detections` array is rendered as a bounding box on the camera feed.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `bbox` | `number[4]` | ✅ | `[x, y, width, height]` in pixels |
| `class_name` | `string` | ✅ | Label text shown on the bounding box |
| `confidence` | `number` | ✅ | Confidence score (0-1, displayed as 0-100%) |
| `class_id` | `number` | ❌ | Numeric class ID (displayed next to label) |
| `grade` | `string` | ❌ | Quality grade (e.g. "premium", "grade_a", "defective") |

### Bounding Box Color Coding

Confidence thresholds for color rendering (client-side):

| Confidence | Color |
|-----------|-------|
| ≥ 80% | `#34C759` (green) |
| ≥ 50% | `#FF9500` (orange) |
| < 50% | `#FF3B30` (red) |

### Tenant Icon Mapping

The app also shows a domain icon per detection based on the `tenantType` from QR:

| tenantType | Icon | Color |
|-----------|------|-------|
| `agriculture` | Apple (green) | `#34C759` |
| `waste_management` | Trash2 (blue) | `#007AFF` |
| `warehouse` | Package (orange) | `#FF9500` |
| _other_ | Camera (gray) | `#8E8E93` |

## Authentication

CamLiveAI uses **Bearer token authentication**.

1. The `token` field from the QR code is sent as `Authorization: Bearer <token>`
2. The backend validates this token (JWT or opaque API key)
3. If the token is invalid, return `401 Unauthorized` — CamLiveAI logs and continues

### Token Types

**JWT (recommended):**
- Embed `sub` (user ID), `tenantType`, `cameraId` in claims
- Sign with your secret
- Set an expiry (e.g. 30 days) — the QR code is static, so plan for refresh

**Opaque API Key:**
- Store key → user mapping on your backend
- Simpler but harder to revoke

### Token Requirements

Your token endpoint needs to map the token to:
- A valid user/device
- The correct `tenantType` for domain-aware filtering
- The correct `tenantId` for data isolation

> **Pre-existing tokens:** You don't need to modify your auth system. If you already
> have API tokens, just put them in the QR code `token` field and validate on each request.

## Generating QR Codes

### Python Example

```python
import qrcode
import json

qr_data = json.dumps({
    "token": "your-jwt-or-api-token",
    "tenantType": "agriculture",
    "tenantSlug": "fruitsight",
    "cameraId": "cam-001",
    "apiUrl": "https://fruitsight-ai.onrender.com",
})

qr = qrcode.QRCode(
    version=4,
    error_correction=qrcode.constants.ERROR_CORRECT_M,
    box_size=10,
    border=4,
)
qr.add_data(qr_data)
qr.make(fit=True)

img = qr.make_image(fill_color="black", back_color="white")
img.save("camliveai-qr.png")
```

### Node.js Example

```javascript
import QRCode from 'qrcode';

const data = JSON.stringify({
  token: 'your-jwt-or-api-token',
  tenantType: 'agriculture',
  tenantSlug: 'fruitsight',
  cameraId: 'cam-001',
  apiUrl: 'https://fruitsight-ai.onrender.com',
});

await QRCode.toFile('camliveai-qr.png', data, {
  errorCorrectionLevel: 'M',
  type: 'png',
  width: 400,
});
```

### Browser Example (generate from your web app)

```javascript
import QRCode from 'qrcode';

async function generateCameraQR(cameraId, token, tenantType, tenantSlug, apiUrl) {
  const data = JSON.stringify({
    token,
    tenantType,
    tenantSlug,
    cameraId,
    apiUrl,
  });

  // Display as image
  const qrCanvas = document.getElementById('qr-canvas');
  await QRCode.toCanvas(qrCanvas, data, {
    errorCorrectionLevel: 'M',
    width: 300,
    margin: 2,
  });

  // Or get as data URL
  const qrDataUrl = await QRCode.toDataURL(data, {
    errorCorrectionLevel: 'M',
    width: 300,
  });

  return qrDataUrl;
}
```

### CLI with `qrencode`

```bash
qrencode -o camliveai-qr.png -l M -s 10 \
  '{"token":"...","tenantType":"agriculture","tenantSlug":"fruitsight","cameraId":"cam-001","apiUrl":"https://fruitsight-ai.onrender.com"}'
```

## Registering Users

To use CamLiveAI with the FruitSight AI backend (or any backend with auth), you need:

1. **A registered user account** that the token belongs to
2. **A valid JWT token** for that user

### Example: Register & Get Token

```bash
# 1. Register a user
curl -X POST https://your-api.com/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "camera-user@example.com",
    "password": "your-password",
    "name": "Warehouse Camera 1",
    "tenantType": "agriculture"
  }'

# 2. Login to get a JWT token
curl -X POST https://your-api.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "camera-user@example.com",
    "password": "your-password"
  }'
# Response: { "token": "eyJhbGciOi...", ... }

# 3. Use that token in your QR code
```

### Required Fields for Registration

| Field | Type | Description |
|-------|------|-------------|
| `email` | `string` | User email (unique) |
| `password` | `string` | Password (min 6 chars) |
| `name` | `string` | Display name |
| `tenantType` | `string` | One of: `agriculture`, `waste_management`, `warehouse`, `universal` |

## Domain / Tenant Types

These are the standard domain identifiers. CamLiveAI uses the `tenantType` from the QR
code to:
- Map to the correct model on the HF Space backend
- Apply domain-specific class filtering
- Display the correct icon in the detection feed

| tenantType | Project | Classes Detected |
|-----------|---------|-----------------|
| `agriculture` | FruitSight AI | pineapple, cherry, tomato, watermelon |
| `waste_management` | WasteSight AI | bottle, can, plastic, paper, organic, other |
| `warehouse` | InventorySight AI | forklift, pallet, box, pallet_truck, stillage, person, robot, equipment |
| `universal` | UniversalSight AI | All of the above (super admin) |

### Custom Domains

If your project has a new domain:
1. Set `tenantType` to a custom string (e.g. `"retail"`, `"security"`)
2. Your backend handles detection however you like
3. CamLiveAI will display a generic camera icon for unknown domains

## Testing Your Integration

### 1. Verify your API returns detections

```bash
# Encode a test image
IMAGE_B64=$(base64 -w0 test-image.jpg)

curl -X POST https://your-api.com/detection/single \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "{\"image\": \"$IMAGE_B64\", \"confidence\": 0.5}"
```

Expected response:
```json
{
  "detections": [
    {
      "bbox": [100, 50, 200, 180],
      "class_name": "apple",
      "confidence": 0.92
    }
  ],
  "total_count": 1,
  "processing_time_ms": 1620.5
}
```

### 2. Generate a QR code with a known token

Use any of the [QR generation examples](#generating-qr-codes) above.

### 3. Test the full flow

1. Open `https://camliveai.netlify.app` on your phone
2. Hold the QR code in front of the camera
3. CamLiveAI should auto-detect, switch to camera, and start sending frames
4. Point the camera at objects — bounding boxes should appear

### 4. Check the console

Open Chrome DevTools on your phone (via `chrome://inspect`) to see:
- API request/response logs
- Token validation errors
- CORS errors (if any)

### CORS

Make sure your backend includes these CORS headers:

```
Access-Control-Allow-Origin: https://camliveai.netlify.app
Access-Control-Allow-Methods: POST, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
```

Example (FastAPI):
```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://camliveai.netlify.app"],
    allow_methods=["POST", "OPTIONS"],
    allow_headers=["Content-Type", "Authorization"],
)
```

## Reference: CamLiveAI Internals

### How Frames Are Captured

```typescript
// src/hooks/useRealtimeDetection.ts
const captureFrame = () => {
  canvas.width = video.videoWidth || 640;
  canvas.height = video.videoHeight || 480;
  ctx.drawImage(video, 0, 0, canvas.width, canvas.height);
  const dataUrl = canvas.toDataURL('image/jpeg', 0.7);
  return dataUrl.replace(/^data:image\/jpeg;base64,/, '');
};
```

- Resolution: matches the video stream (typically 640×480 to 1920×1080)
- Quality: 0.7 (JPEG compression)
- Interval: configurable (3s single / 500ms live / 5s batch)

### Data Model

```typescript
// src/types.ts
interface Detection {
  id: string;
  bbox: number[];        // [x, y, w, h]
  class_name: string;
  confidence: number;    // 0-100 (converted from 0-1 on receipt)
  class_id?: number;
  grade?: string;
}

interface PairingConfig {
  token: string;
  tenantType: string;
  tenantSlug: string;
  cameraId: string;
  apiUrl: string;
}
```

### Detection Feed

- Stores up to **50 recent detections** in memory
- Shows confidence as percentage with green (≥80%), orange (≥50%), red (<50%) colors
- Downloadable as JSON via the feed UI
- Domain icon per detection based on `tenantType`

### Settings

Users can configure:
- **Camera Source**: Default / front / back
- **Resolution**: 640×480 / 1280×720 / 1920×1080
- **FPS Target**: 15 / 30 / 60
- **Confidence Threshold**: slider 0-100 (client-side filter)
- **Show Bounding Boxes**: toggle
- **Show Labels**: toggle
- **Show Grid Overlay**: toggle
- **Night Mode**: toggle
