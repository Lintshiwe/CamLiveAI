# CamLiveAI QR Code Specification

This document defines the QR code format that CamLiveAI accepts. Any project can generate a QR code following this spec, and CamLiveAI will automatically scan, parse, and connect to it.

## QR Code Data Format

The QR code must contain a **JSON object** with the following fields:

```json
{
  "token": "jwt_or_api_key_here",
  "tenantType": "agriculture",
  "tenantSlug": "my-farm",
  "cameraId": "cam-001",
  "apiUrl": "https://your-api.com"
}
```

### Field Reference

| Field | Required | Type | Description |
|-------|----------|------|-------------|
| `token` | ✅ Yes | string | Authentication token (JWT or API key) for the detection API |
| `tenantType` | ✅ Yes | string | Domain type: `agriculture`, `waste_management`, `warehouse`, or `general` |
| `tenantSlug` | ❌ No | string | Human-readable tenant identifier (e.g., `my-farm`, `waste-plant`) |
| `cameraId` | ✅ Yes | string | Unique camera/device identifier generated by the platform |
| `apiUrl` | ❌ No | string | Custom API endpoint URL (defaults to `https://fruitsight-ai.onrender.com` if omitted) |

## How to Generate a QR Code

### Option 1: Using a QR Code Library (Recommended)

```javascript
// Node.js / Browser
import QRCode from 'qrcode';

const data = {
  token: "your-auth-token",
  tenantType: "agriculture",
  tenantSlug: "your-tenant",
  cameraId: "CAM-001",
  apiUrl: "https://your-api.com"
};

QRCode.toDataURL(JSON.stringify(data), { width: 400 });
```

### Option 2: Via a QR Code Generator API

```
https://api.qrserver.com/v1/create-qr-code/?size=400x400&data={...}
```

Replace `{...}` with the URL-encoded JSON string.

### Option 3: Integrate into Your Website

Add a "Generate QR Code" button in your camera/detection dashboard:

```jsx
const generateQR = async (cameraId, token, tenant) => {
  const qrData = {
    token,
    tenantType: tenant.type,
    tenantSlug: tenant.slug,
    cameraId,
    apiUrl: window.location.origin
  };

  // Generate QR as image
  const qrImage = await QRCode.toDataURL(JSON.stringify(qrData));
  
  // Display or print
  document.getElementById('qrDisplay').src = qrImage;
};
```

## Connection Flow

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Your App/  │     │   CamLiveAI      │     │  Detection API  │
│  Website    │     │   (Phone)        │     │  (Backend)      │
└──────┬──────┘     └────────┬─────────┘     └────────┬────────┘
       │                     │                        │
       │  Generate QR        │                        │
       │  (JSON encoded)     │                        │
       │                     │                        │
       │─── Display QR ─────▶│                        │
       │                     │                        │
       │                     │  Auto-Scan QR          │
       │                     │  Parse JSON            │
       │                     │  Extract config        │
       │                     │                        │
       │                     │  POST /detection/single │
       │                     │  (with token + image)   │
       │                     │◀─────── Results ────────│
       │                     │                        │
       │                     │  Display detections    │
       │                     │  (bbox overlays, list) │
       │                     │                        │
```

## Security Notes

- The `token` field should be a short-lived JWT or scoped API key
- Never include sensitive data (passwords, private keys) in the QR code
- The QR code is scanned locally on the device — data never leaves the phone until the API call is made
- Use HTTPS for all `apiUrl` endpoints to encrypt data in transit

## Compatibility

Any project that generates a QR code matching this JSON schema will work with CamLiveAI automatically. The app:
1. Auto-detects the QR code via camera
2. Parses the JSON
3. Converts `tenantType` to appropriate UI icons and colors
4. Starts sending frames to the provided `apiUrl` with the `token` for auth
5. Displays detection results in real-time
